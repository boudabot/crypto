# Docs Organization

## Purpose
This folder is organized by document type so the active source of truth is easy to find quickly.

## Structure

### `docs/active/`
Current implementation source of truth.

### `docs/reference/`
Secondary context that can help but should not override active docs.

### `docs/process/`
Execution and collaboration instructions.

### `docs/ui/`
UI direction, screenshots, and external design inspirations.

## Recommendation
Yes, the repo should keep both:

- a `docs/ui/screenshots/` folder for local visual targets
- a `docs/ui/DESIGN_INSPIRATIONS.md` file for external URLs

They serve different needs and work best together.
