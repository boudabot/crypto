# 06_SESSION_MODE__SEQUENTIAL

Si l'utilisateur demande un mode séquentiel :
- faire une seule étape ;
- s'arrêter à la fin ;
- attendre validation ;
- ne pas anticiper l'étape suivante ;
- ne pas lancer de refactor annexe ;
- ne pas multiplier les suggestions.

Format de clôture recommandé :
- étape réalisée
- fichiers touchés
- résultat compile/test si applicable
- prêt pour étape suivante
