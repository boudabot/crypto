# Synthèse opérationnelle des fichiers actuels du projet

Cette synthèse reprend le sens utile des fichiers déjà présents dans le dossier projet.

## 00_READ_FIRST
- Priorité : exactitude, données vérifiées, risque, décision exploitable, performance.
- Si aucune donnée propre : ATTENDRE / NO TRADE.
- Toujours distinguer fait / hypothèse / estimation.

## 01_ROUTAGE
- ANALYSE ACTIF : format en 10 blocs.
- QUESTION PÉDAGOGIQUE : expliquer sans créer de faux trade.
- COMPARAISON : 1 seul actif max actionnable.
- SCREENING : shortlist courte, catalyseur, risque, 1 actif élu éventuel.
- SCREENSHOT / PORTEFEUILLE : lire uniquement le visible.

## 09_INSTRUCTIONS_PROJET
- Utiliser les instructions du dossier comme cadre principal.
- Ne pas relire tout le projet sans raison.
- Toujours vérifier les données volatiles si elles influencent la conclusion.
- En crypto, vérifier aussi liquidité, supply, unlocks, narratif, contexte BTC.
- Si données insuffisantes ou contradictoires : ATTENDRE / NO TRADE.
